# The Discoverability Stack Framework

See THESIS.md for the full framework. This file tracks detailed notes on each layer.

## Layer 1: Machine-Readability
- MCP schemas, OpenAPI specs, tool descriptions
- Quality of documentation
- Structured metadata (capabilities, pricing, limitations)
- **Key question:** Can an agent programmatically understand what you do?

## Layer 2: Visibility
- App store listings (ChatGPT, Claude, etc.)
- Registry presence (MCP registries, API directories)
- Category rankings, keyword optimization
- **Key question:** Can agents find you when browsing?

## Layer 3: Evaluation
- Performance benchmarks (uptime, speed, accuracy)
- User reviews and ratings
- Cost-effectiveness metrics
- Capability-to-need matching scores
- **Key question:** When compared to alternatives, do you win?

## Layer 4: Intent Matching (Organic/Prompt Discovery)
- Does an LLM recommend you when asked about your category?
- Are you in the training data? In RAG sources?
- Do agents select you for the right prompts?
- **Key question:** Does AI choose you when someone has the intent?
